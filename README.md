# מסע בארץ ישראל

אפליקציית למידה אינטראקטיבית בגאוגרפיה של ארץ ישראל, לכיתה ט׳.
HTML/CSS/JS בלבד, ללא שרת, רצה כמו שהיא ב־GitHub Pages.

## מה קיים כרגע (שלבים 1–2)

- **מסך פתיחה** (`index.html`) — שם תלמיד, התחלה, המשך, איפוס, נגישות.
- **מסך מסלול** (`pages/route.html`) — שש תחנות עם ארבעה מצבים, כרטיס תחנה, פס התקדמות כולל.
- **תשתית** — שמירה ב־localStorage, היגיון התקדמות ונעילה, נגישות, ניווט.
- **מצב מורה** (`teacher/teacher.html`) — פתיחת כל התחנות, רמזים, הצגת ניקוד, מעבר ישיר, איפוס.

## מה עוד לא נבנה

תבנית התחנה (שלב 3), חמש התחנות, משימת הסיכום, מסך התוצאות, והמפה האינטראקטיבית.
בהתאם, `js/map.js` ו־`js/feedback.js` ייווצרו בשלבים הבאים.

## הרצה מקומית

אין תהליך build. פותחים את `index.html` בדפדפן, או:

```bash
python3 -m http.server 8000
```

## כללי עבודה לדפי התחנות (שלב 3 והלאה)

כל דף תחנה חדש:

1. `<html lang="he" dir="rtl" data-root="..">` — משם נגזרים כל הנתיבים.
2. טוען לפי הסדר: `data/stations.js`, `storage.js`, `progress.js`, `navigation.js`, `accessibility.js`, ואז הסקריפט של התחנה.
3. קורא ל־`IGJ.accessibility.init()` פעם אחת.
4. שומר התקדמות דרך `IGJ.progress.markStep(id, step)` — לא ניגש ל־localStorage ישירות.
5. מסיים ב־`IGJ.progress.completeStation(id, score)` — זה מה שפותח את התחנה הבאה.

הוספת תחנה = הוספת פריט ל־`data/stations.js` + דף ב־`pages/`. מסך המסלול מתעדכן לבד.

## שמירת נתונים

הכול ב־localStorage של הדפדפן, ללא התחברות:

- `igj.state.v1` — התקדמות התלמיד
- `igj.settings.v1` — הגדרות המורה
- `igj.a11y.v1` — העדפות נגישות
